# Лабораторная работа №8. Введение в React + JSX

<img width="653" height="372" alt="image" src="https://github.com/user-attachments/assets/b6f0bc75-f18d-4529-9f21-c7eb76759da4" />

Приложение позволяет:
- Добавлять задачи
- Удалять отдельные задачи
- Очищать весь список задач
- Сохранять задачи в localStorage, чтобы они не пропадали при перезагрузке страницы

## Компоненты
1. **App.jsx** – главный компонент, хранит массив задач в состоянии и управляет функциями добавления, удаления и очистки задач.  
2. **AddTaskForm.jsx** – форма для добавления новой задачи. Получает функцию `addTask` через пропсы.  
3. **ToDoList.jsx** – список задач. Получает массив задач `tasks` и функцию `removeTask` через пропсы.  
4. **ToDoItem.jsx** – отдельная задача с текстом и кнопкой «Удалить».  

## Основные понятия
- **Компонент** – изолированная и переиспользуемая часть интерфейса, которая возвращает JSX.  
- **Пропсы (Props)** – способ передавать данные от родительского компонента к дочернему. Пропсы неизменяемы внутри компонента.  
- **Состояние (State)** – данные, которые могут изменяться внутри компонента. React автоматически перерисовывает компонент при изменении состояния.  

## Работа с ИИ
Большую часть кода компонентов с функционалом добавления, удаления и очистки задач с использованием `useState` я сгенерировала с помощью ИИ, чтобы получить рабочий шаблон (boilerplate).  

## Вывод о компонентном подходе
Компонентный подход позволяет:
- Декларативно описывать интерфейс в зависимости от данных  
- Делить интерфейс на независимые, переиспользуемые блоки  
- Сосредоточиться на логике данных, а не на прямом управлении DOM
